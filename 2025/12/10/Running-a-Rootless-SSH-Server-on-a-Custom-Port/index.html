Running a Rootless SSH Server on a Custom Port

Sometimes you need to spin up an SSH server without changing system configs or needing root. Here’s how you can do it, using a custom port and a host key you generate yourself.

Prerequisites

  • /usr/sbin/sshd properly configured to facilitate logins and ChrootDirectory (usually /var/empty) set up by root.

Note: If /usr/sbin/sshd is not properly configured or you want to reconfigure the logic to facilitate logins (e.g., allow password login), and you don’t have root access, this method won’t work. Consider alternatives like telnet for shell or WebDAV for file transfers.
